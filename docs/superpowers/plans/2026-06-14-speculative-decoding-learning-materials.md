<h2 id="s1">1. Why Autoregressive Decoding Is Slow</h2>

<p>A transformer language model generates text one token at a time. To generate \(K\) tokens:</p>
<ol>
  <li>Feed the full current sequence to the model (forward pass).</li>
  <li>Get logits for the next token position.</li>
  <li>Sample or argmax to select the next token.</li>
  <li>Append that token and repeat — \(K\) times total.</li>
</ol>

<p>This requires \(K\) <strong>serial</strong> forward passes. Token \(i+1\) cannot be computed until token \(i\) is known, because the model is conditioned on the full prefix.</p>

<h3>Memory Bandwidth Bottleneck</h3>

<p>Modern LLMs are not compute-bound during inference — they are <strong>memory bandwidth-bound</strong>. Each forward pass must load the entire model weight matrix from GPU High Bandwidth Memory (HBM) to compute units. For a model with \(N\) parameters in 16-bit precision, each forward pass moves \(2N\) bytes through the memory bus. A single token generation step does \(\mathcal{O}(N)\) multiply-adds but transfers \(\mathcal{O}(N)\) bytes — so the arithmetic units are starved waiting for data.</p>

<p>Consequence: the walltime for one forward pass is roughly <em>constant</em> regardless of batch size (up to a point). Running \(\gamma + 1\) sequences in a batch takes roughly the same time as running 1 — the extra arithmetic is "free" because arithmetic units were idle.</p>

<div class="keyfacts">
<strong>Key Facts — Section 1</strong>
<ul>
  <li>K tokens requires K serial forward passes. No shortcut without architecture changes.</li>
  <li>Inference is memory-bandwidth-bound, not compute-bound. Arithmetic units sit idle.</li>
  <li>Batching \(\gamma+1\) sequences costs roughly the same walltime as batching 1 — this is the foundation of speculative decoding's speedup.</li>
</ul>
</div>

<h2 id="s2">2. Speculative Execution</h2>

<p><strong>Speculative execution</strong> is a classic CPU optimization: perform a task before you know if it's needed, betting that it will be. The canonical example is branch prediction — the CPU predicts which branch will be taken and begins executing it; if wrong, it discards the work.</p>

<p>The paper generalizes this to the <em>stochastic</em> setting: a task might be needed with some probability \(p\). The core idea applied to LLMs:</p>

<ol>
  <li>Use a cheap <strong>draft model</strong> \(M_q\) to guess \(\gamma\) likely next tokens.</li>
  <li>Run the expensive <strong>target model</strong> \(M_p\) in parallel on all \(\gamma+1\) possible prefixes.</li>
  <li>Accept or reject draft tokens in a way that guarantees the final output distribution equals \(M_p\)'s distribution exactly.</li>
</ol>

<p>The bet: \(M_q\) is usually right, so \(M_p\)'s single batched pass generates multiple tokens instead of one.</p>

<div class="keyfacts">
<strong>Key Facts — Section 2</strong>
<ul>
  <li>Speculative execution: do work before you know it's needed, discard if wrong.</li>
  <li>The draft model guesses; the target model verifies. Output distribution is <em>unchanged</em>.</li>
  <li>Speedup comes from the target model's batch being nearly "free" compared to running it \(\gamma+1\) times serially.</li>
</ul>
</div>

<h2 id="s3">3. Standardized Sampling</h2>

<p>The paper's algorithm only needs to reason about probability distributions \(p(x)\) and \(q(x)\) over the vocabulary. All practical sampling strategies can be cast into this framework by adjusting the logits before softmax:</p>

<ul>
  <li><strong>Greedy / argmax:</strong> Zero all non-maximum logits, normalize. The resulting distribution is a point mass on the argmax token.</li>
  <li><strong>Top-k:</strong> Zero all but the top-\(k\) logits, normalize.</li>
  <li><strong>Nucleus (top-p):</strong> Zero all tokens outside the smallest set whose cumulative probability exceeds \(p\), normalize.</li>
  <li><strong>Temperature \(T\):</strong> Divide logits by \(T\) before softmax. \(T \to 0\) approaches argmax; \(T \to \infty\) approaches uniform.</li>
</ul>

<p>After applying any of these transformations, we have a valid probability distribution. The algorithm treats \(p(x)\) and \(q(x)\) as the <em>already-adjusted</em> distributions.</p>

<div class="keyfacts">
<strong>Key Facts — Section 3</strong>
<ul>
  <li>All sampling strategies reduce to sampling from an adjusted distribution over the vocabulary.</li>
  <li>\(p(x)\) and \(q(x)\) in the algorithm are post-adjustment distributions, not raw logits.</li>
  <li>Draft and target must use the <em>same</em> sampling strategy (both use the same temperature, top-k, etc.) for the algorithm to be valid.</li>
</ul>
</div>

<h2 id="s4">4. Algorithm 1 — Line-by-Line</h2>

<h3>Notation</h3>
<ul>
  <li>\(M_p\): target model (expensive, the one we want to sample from)</li>
  <li>\(M_q\): draft model (cheap approximation, same task)</li>
  <li>\(\gamma\): number of draft tokens to speculate (hyperparameter)</li>
  <li>\(p_i(x) = p(x \mid \text{prefix} + [x_1, \ldots, x_{i-1}])\): target distribution at position \(i\)</li>
  <li>\(q_i(x)\): draft distribution at position \(i\)</li>
  <li>\(\text{norm}(v)\): normalize vector \(v\) to sum to 1</li>
</ul>

<div class="algo">SpeculativeDecodingStep(Mp, Mq, prefix):

  // Phase 1: Draft model generates γ tokens autoregressively
  for i = 1 to γ:
      q_i(x) ← Mq(prefix + [x₁, ..., x_{i-1}])   // draft distribution
      x_i ~ q_i(x)                                   // sample draft token

  // Phase 2: Target model evaluates all γ+1 prefixes IN PARALLEL
  p₁(x), ..., p_{γ+1}(x) ← Mp(prefix),
                              Mp(prefix + [x₁]),
                              Mp(prefix + [x₁, x₂]),
                              ...,
                              Mp(prefix + [x₁, ..., x_γ])

  // Phase 3: Rejection loop
  r₁ ~ U(0,1), ..., r_γ ~ U(0,1)
  n ← min({ i-1 | r_i > p_i(x_i)/q_i(x_i) } ∪ {γ})

  // Phase 4: Adjust distribution (only if a token was rejected)
  p'(x) ← p_{n+1}(x)
  if n < γ:
      p'(x) ← norm(max(0, p_{n+1}(x) - q_{n+1}(x)))

  // Phase 5: Sample final token and return
  t ~ p'(x)
  return prefix + [x₁, ..., x_n, t]
</div>

<h3>Step-by-Step Explanation</h3>

<p><strong>Phase 1 (Draft):</strong> The draft model generates \(\gamma\) tokens autoregressively. This is sequential but cheap. Each token \(x_i\) is sampled from \(q_i\), and the draft records the distribution \(q_i(x)\) at each step.</p>

<p><strong>Phase 2 (Parallel target evaluation):</strong> The target model is called with a <em>batch</em> of \(\gamma+1\) sequences. The batch is:
\[ [\text{prefix}], [\text{prefix}, x_1], [\text{prefix}, x_1, x_2], \ldots, [\text{prefix}, x_1, \ldots, x_\gamma] \]
This costs roughly one target forward pass (sequences differ only in length by 1 token, and the memory bandwidth bottleneck means batching is nearly free).</p>

<p><strong>Phase 3 (Rejection):</strong> For each draft token \(x_i\), we accept it with probability \(\min(1, p_i(x_i) / q_i(x_i))\). We draw a uniform \(r_i \sim U(0,1)\) and accept iff \(r_i \leq p_i(x_i)/q_i(x_i)\). The variable \(n\) is the index of the first rejection minus one (i.e., \(n\) tokens were accepted). If no rejection, \(n = \gamma\).</p>

<p><strong>Phase 4 (Adjusted distribution):</strong> If any token was rejected (\(n < \gamma\)), we sample a <em>correction token</em> from the adjusted distribution \(p'(x) = \text{norm}(\max(0, p_{n+1}(x) - q_{n+1}(x)))\). This compensates for the bias introduced by the draft model having proposed the wrong token. If all tokens were accepted (\(n = \gamma\)), we sample a <em>bonus token</em> from \(p_{n+1}(x)\) — the target distribution after the last accepted draft token.</p>

<p><strong>Why does this produce exactly \(p\)'s distribution?</strong> See Section 5. The acceptance-rejection scheme ensures each output token follows \(M_p\)'s distribution exactly, regardless of \(M_q\).</p>

<p><strong>Guaranteed minimum:</strong> We always produce at least 1 new token per call (the correction or bonus token from \(t \sim p'(x)\)). In the worst case (\(n=0\), first draft token rejected), we get exactly 1 token — same as autoregressive decoding. In the best case (\(n=\gamma\)), we get \(\gamma+1\) tokens.</p>

<div class="keyfacts">
<strong>Key Facts — Section 4</strong>
<ul>
  <li>Draft phase is sequential and cheap; target phase is one batched call (nearly free).</li>
  <li>Each step produces 1 to \(\gamma+1\) tokens. Worst case = autoregressive decoding.</li>
  <li>When a draft token is rejected, all subsequent draft computations for that step are discarded.</li>
  <li>We always append exactly one token from the target model's distribution — either a correction (if rejected) or a bonus (if all accepted).</li>
</ul>
</div>

<h2 id="s5">5. Speculative Sampling — Full Mathematical Treatment</h2>

<h3>The Core Claim</h3>

<p>We want to sample \(x \sim p(x)\) but only have efficient access to \(q(x)\). Speculative sampling does this:</p>
<ol>
  <li>Sample \(x \sim q(x)\)</li>
  <li>Accept \(x\) with probability \(\min\!\left(1, \frac{p(x)}{q(x)}\right)\)</li>
  <li>If rejected, sample \(x\) from \(p'(x) = \text{norm}(\max(0, p(x) - q(x)))\)</li>
</ol>

<div class="theorem">
<strong>Claim:</strong> The output of this procedure is distributed as \(p(x)\).
</div>

<div class="proof-toggle" onclick="toggleProof('proof-spec-samp')">▶ Proof (click to expand)</div>
<div class="proof-body" id="proof-spec-samp">
<p>Let \(\alpha = \sum_x \min(p(x), q(x))\) be the probability that a draft sample is accepted. Then:</p>

<p>\[P(\text{accept}) = \sum_x q(x) \cdot \min\!\left(1, \frac{p(x)}{q(x)}\right) = \sum_x \min(q(x), p(x)) = \alpha\]</p>

<p>The output token \(t\) satisfies:</p>
\[P(t = x) = \underbrace{q(x) \cdot \min\!\left(1, \frac{p(x)}{q(x)}\right)}_{\text{draft accepted}} + \underbrace{(1-\alpha) \cdot p'(x)}_{\text{resampled after rejection}}\]

<p>The first term simplifies: \(q(x) \cdot \min(1, p(x)/q(x)) = \min(q(x), p(x))\).</p>

<p>For the second term, \(p'(x) = \frac{\max(0, p(x) - q(x))}{1 - \alpha}\) (normalization constant is \(\sum_x \max(0, p(x)-q(x)) = \sum_x p(x) - \min(p(x), q(x)) = 1 - \alpha\)).</p>

<p>So \((1-\alpha) \cdot p'(x) = \max(0, p(x) - q(x))\).</p>

<p>Therefore:
\[P(t = x) = \min(q(x), p(x)) + \max(0, p(x) - q(x))\]</p>

<p>Case 1: \(p(x) \leq q(x)\). Then \(\min = p(x)\), \(\max = 0\). Sum \(= p(x)\). ✓</p>
<p>Case 2: \(p(x) > q(x)\). Then \(\min = q(x)\), \(\max = p(x) - q(x)\). Sum \(= p(x)\). ✓</p>

<p>In both cases \(P(t = x) = p(x)\). \(\blacksquare\)</p>
</div>

<h3>The DLK Divergence</h3>

<div class="definition">
<strong>Definition 3.2 (DLK divergence):</strong>
\[D_{LK}(p, q) = \sum_x \left|q(x) - M(x)\right| \quad \text{where } M(x) = \frac{p(x) + q(x)}{2}\]
</div>

<div class="theorem">
<strong>Lemma 3.3:</strong> \(D_{LK}(p, q) = 1 - \sum_x \min(p(x), q(x))\)
</div>

<div class="proof-toggle" onclick="toggleProof('proof-dlk')">▶ Proof (click to expand)</div>
<div class="proof-body" id="proof-dlk">
<p>\[D_{LK}(p,q) = \sum_x \left|q(x) - \frac{p(x)+q(x)}{2}\right| = \sum_x \frac{|p(x)-q(x)|}{2}\]</p>
<p>\[= \frac{1}{2}\sum_x (p(x) + q(x) - 2\min(p(x),q(x))) = \frac{(1+1) - 2\sum_x\min(p(x),q(x))}{2} = 1 - \sum_x \min(p(x),q(x))\]</p>
<p>\(\blacksquare\)</p>
</div>

<div class="theorem">
<strong>Corollary 3.4:</strong> \(D_{LK}(p,q) \in [0,1]\), symmetric. \(D_{LK}=0 \Leftrightarrow p=q\). \(D_{LK}=1 \Leftrightarrow\) disjoint support.
</div>

<div class="definition">
<strong>Definition 3.1 (Acceptance rate):</strong> \(\beta_{x_{<t}} = P(\text{accept draft token} \mid \text{prefix } x_{<t})\)
</div>

<div class="theorem">
<strong>Theorem 3.5:</strong> \(\beta = 1 - D_{LK}(p, q)\)
</div>

<div class="proof-toggle" onclick="toggleProof('proof-beta')">▶ Proof (click to expand)</div>
<div class="proof-body" id="proof-beta">
<p>\[\beta = \mathbb{E}_{x \sim q}\!\left[\min\!\left(1, \frac{p(x)}{q(x)}\right)\right] = \sum_x q(x) \cdot \min\!\left(1, \frac{p(x)}{q(x)}\right) = \sum_x \min(p(x), q(x))\]</p>
<p>By Lemma 3.3, \(\sum_x \min(p(x), q(x)) = 1 - D_{LK}(p,q)\). \(\blacksquare\)</p>
</div>

<div class="theorem">
<strong>Corollary 3.6:</strong> \(\alpha = \mathbb{E}[\beta] = \mathbb{E}[\min(p, q)]\)
</div>

<p>This is the key metric: \(\alpha\) summarizes how well \(M_q\) approximates \(M_p\) across typical prefixes. \(\alpha \to 1\) means the draft is almost always accepted; \(\alpha \to 0\) means speculative decoding degenerates to autoregressive.</p>

<div class="keyfacts">
<strong>Key Facts — Section 5</strong>
<ul>
  <li>Speculative sampling is exact: output distribution equals \(p(x)\) regardless of \(q(x)\).</li>
  <li>The adjusted distribution \(p'(x) = \text{norm}(\max(0, p-q))\) compensates for the draft model's bias.</li>
  <li>\(\alpha = \mathbb{E}[\sum_x \min(p(x), q(x))]\): the expected probability mass where draft and target agree.</li>
  <li>\(\alpha = 1 - \mathbb{E}[D_{LK}(p,q)]\): higher divergence between models → lower acceptance rate → less speedup.</li>
</ul>
</div>

<h2 id="s6">6. Analysis</h2>

<h3>Expected Tokens Per Iteration</h3>

<p>Let \(\alpha = \mathbb{E}[\beta]\) (mean acceptance rate, assumed i.i.d. per position). In one call to Algorithm 1, the number of accepted draft tokens follows a <em>capped geometric distribution</em>: we accept each token independently with probability \(\alpha\), and stop at the first rejection or at \(\gamma\). Plus one bonus/correction token always.</p>

<p>Let \(N\) = number of accepted draft tokens (before the correction/bonus). Then \(N \in \{0, 1, \ldots, \gamma\}\) with:
\[P(N = k) = \begin{cases} \alpha^k (1-\alpha) & k < \gamma \\ \alpha^\gamma & k = \gamma \end{cases}\]</p>

<p>Total tokens per step = \(N + 1\) (draft tokens accepted + 1 correction/bonus). Thus:</p>

<div class="theorem">
<strong>Equation 1:</strong>
\[\mathbb{E}[\text{tokens per step}] = 1 + \mathbb{E}[N] = \frac{1 - \alpha^{\gamma+1}}{1 - \alpha}\]
</div>

<div class="proof-toggle" onclick="toggleProof('proof-tokens')">▶ Derivation (click to expand)</div>
<div class="proof-body" id="proof-tokens">
<p>\[\mathbb{E}[N] = \sum_{k=0}^{\gamma-1} k \cdot \alpha^k(1-\alpha) + \gamma \cdot \alpha^\gamma\]</p>
<p>Let \(S = \sum_{k=0}^\gamma k \cdot \alpha^k (1-\alpha)\) (geometric series derivative):</p>
<p>\[\sum_{k=0}^{\gamma} \alpha^k = \frac{1 - \alpha^{\gamma+1}}{1-\alpha}\]</p>
<p>\[\mathbb{E}[N+1] = \mathbb{E}[N] + 1 = \sum_{k=0}^{\gamma}(k+1) P(N=k) = \sum_{j=1}^{\gamma+1} j \cdot P(N=j-1)\]</p>
<p>Equivalently: \(\mathbb{E}[N+1] = \sum_{k=0}^\gamma P(N \geq k) = \sum_{k=0}^\gamma \alpha^k = \frac{1-\alpha^{\gamma+1}}{1-\alpha}\). \(\blacksquare\)</p>
</div>

<h3>Walltime Improvement</h3>

<div class="definition">
<strong>Definition 3.7 (Cost coefficient):</strong> \(c = \frac{\text{walltime of one } M_q \text{ run}}{\text{walltime of one } M_p \text{ run}}\)
</div>

<p>In practice, \(M_q\) is orders of magnitude smaller than \(M_p\), so \(c \ll 1\) (often \(c < 0.05\)).</p>

<div class="theorem">
<strong>Theorem 3.8 (Walltime speedup):</strong>
\[\text{speedup} = \frac{1 - \alpha^{\gamma+1}}{(1-\alpha)(\gamma c + 1)}\]
</div>

<div class="proof-toggle" onclick="toggleProof('proof-walltime')">▶ Proof (click to expand)</div>
<div class="proof-body" id="proof-walltime">
<p>Let \(T\) = walltime of one \(M_p\) forward pass. Each call to Algorithm 1 costs:</p>
<ul>
  <li>\(\gamma c T\): running \(M_q\) for \(\gamma\) steps</li>
  <li>\(T\): one batched \(M_p\) call (the \(\gamma+1\) parallel evaluations cost ~1 pass due to batching)</li>
</ul>
<p>Total cost per step: \((\gamma c + 1)T\). Expected tokens per step: \(\frac{1-\alpha^{\gamma+1}}{1-\alpha}\).</p>
<p>Cost per token with Algorithm 1: \(\frac{(\gamma c + 1)(1-\alpha)}{1-\alpha^{\gamma+1}} T\).</p>
<p>Cost per token with autoregressive: \(T\). Speedup = ratio. \(\blacksquare\)</p>
</div>

<div class="theorem">
<strong>Corollary 3.9:</strong> If \(\alpha > c\), there exists \(\gamma > 0\) giving speedup. Minimum speedup (at \(\gamma=1\)) is \(\frac{1+\alpha}{1+c}\).
</div>

<h3>Arithmetic Operations</h3>

<div class="theorem">
<strong>Theorem 3.11:</strong> The expected factor increase in arithmetic operations is:
\[\frac{(1-\alpha)(\gamma\hat{c} + \gamma + 1)}{1 - \alpha^{\gamma+1}}\]
where \(\hat{c}\) = ratio of FLOPs per token for \(M_q\) vs \(M_p\).
</div>

<p>Note: walltime speedup and arithmetic savings are different! Speculative decoding can increase total FLOPs slightly (rejected drafts waste compute) while still improving walltime (because of the memory bandwidth bottleneck).</p>

<h3>Optimal γ</h3>

<p>Theorem 3.8 gives speedup as a function of \(\gamma\). Maximizing over \(\gamma\) (treating \(\gamma\) as continuous, differentiating, setting to zero) gives an implicit equation. In practice, the optimal \(\gamma\) is found empirically by sweeping values of \(\gamma\) and measuring walltime.</p>

<p><strong>Worked example:</strong> \(\alpha = 0.75\), \(c = 0.03\).</p>
<ul>
  <li>At \(\gamma=4\): speedup \(= \frac{1-0.75^5}{(1-0.75)(4 \times 0.03 + 1)} = \frac{1-0.237}{0.25 \times 1.12} \approx \frac{0.763}{0.280} \approx 2.73\times\)</li>

## Part 1: Baseline Autoregressive Decoding

### How Transformer Generation Works

A causal language model takes token IDs as input and outputs **logits** — one score per vocabulary token — for each position. To generate text:

1. Feed `input_ids` (shape `[1, seq_len]`) to the model.
2. The model returns `logits` (shape `[1, seq_len, vocab_size]`).
3. Take `logits[:, -1, :]` — the prediction for the *next* token.
4. Apply softmax to get a probability distribution, then sample or argmax.
5. Append the chosen token to `input_ids` and repeat.

This is **autoregressive**: each token is conditioned on all previous tokens, so you cannot compute token `i+1` before token `i` is finalized.

```python
# One forward pass looks like:
outputs = model(input_ids)          # input_ids: [1, seq_len]
logits  = outputs.logits            # [1, seq_len, vocab_size]
next_logits = logits[:, -1, :]      # [1, vocab_size] — predict next token
next_token  = next_logits.argmax(-1, keepdim=True)  # [1, 1]
input_ids   = torch.cat([input_ids, next_token], dim=-1)  # [1, seq_len+1]
```

### KV Cache (acknowledged, not implemented here)

In production, transformers cache key/value attention matrices across steps to avoid recomputing the full sequence every time. We skip this for clarity — our implementation recomputes the full prefix each step.

## Part 2: The Bottleneck & Parallelism

### Memory Bandwidth vs Compute

During autoregressive inference, the GPU's arithmetic units are mostly idle. Why? Because each forward pass must load the entire model's weight matrices from High Bandwidth Memory (HBM) into fast compute registers. For a model with N parameters in fp16, that's 2N bytes per pass. The compute for one token is O(N) multiply-adds — but the data transfer bottleneck means the arithmetic units are starved waiting for data.

**Key consequence:** Running a *batch* of B sequences costs nearly the same walltime as running 1 sequence, as long as B is small enough that the batch fits in memory. The extra arithmetic for the additional sequences is essentially free — the hardware was already idle.

This is the foundation of speculative decoding's speedup: we can run γ+1 target model forward passes (on different prefixes) in a single batched call, for roughly the cost of one.

This is the mathematical heart of the paper. Read this section carefully before writing any code.

### Rejection Sampling Intuition

Suppose we want to sample from distribution `p(x)` but it's expensive. We have a cheap surrogate `q(x)`. Rejection sampling says:

1. Sample a candidate `x ~ q(x)` (cheap).
2. Accept it with probability `min(1, p(x)/q(x))`.
3. If rejected, resample from the *residual* distribution `p'(x) = norm(max(0, p(x) - q(x)))`.

**Why `min(1, p(x)/q(x))`?** When `p(x) >= q(x)`, the draft underestimates how often the target would produce `x` — so we always accept (prob=1). When `p(x) < q(x)`, the draft overestimates — we accept only some of the time, scaled by how much the target agrees.

**Why does this work?** See the proof in the HTML reference (Section 5). The key insight: the accepted sample plus the resampled-from-residual sample combine to exactly give `p(x)`.

def speculative_sample(p_probs, q_probs, draft_token_id):
    p_x = p_probs[draft_token_id].item()
    q_x = q_probs[draft_token_id].item()

    accept_prob = min(1.0, p_x / (q_x + 1e-10))

    if torch.rand(1).item() < accept_prob:
        return draft_token_id, True
    else:
        adjusted = torch.clamp(p_probs - q_probs, min=0.0)
        total = adjusted.sum()
        if total < 1e-10:
            return torch.multinomial(p_probs, 1).item(), False
        adjusted = adjusted / total
        return torch.multinomial(adjusted, 1).item(), False
```

    total_accepted = 0
    total_proposed = 0

    for prompt_ids in prompt_ids_list:

            prev_len = current.shape[-1]
            current  = speculative_decode_step(target, draft, current, gamma, device)

            total_accepted += (n_new - 1)
            total_proposed += gamma

def speculative_decode(target, draft, input_ids, max_new_tokens, gamma=4, device='cuda'):
    target.eval()
    draft.eval()
    eos_token_id = target.config.eos_token_id
    prompt_len   = input_ids.shape[-1]
    current      = input_ids.clone().to(device)

    while (current.shape[-1] - prompt_len) < max_new_tokens:
        prev_len = current.shape[-1]
        current  = speculative_decode_step(target, draft, current, gamma, device)
        new_toks = current[0, prev_len:].tolist()

        if eos_token_id is not None and eos_token_id in new_toks:
            eos_pos = new_toks.index(eos_token_id)
            current = current[:, :prev_len + eos_pos + 1]
            break

    # Truncate if overshot
    if current.shape[-1] > prompt_len + max_new_tokens:
        current = current[:, :prompt_len + max_new_tokens]

    return current
```
