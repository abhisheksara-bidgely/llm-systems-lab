1. Why Autoregressive Decoding Is Slow

A transformer language model generates text one token at a time. To generate \(K\) tokens:

  1. Feed the full current sequence to the model (forward pass).
  2. Get logits for the next token position.
  3. Sample or argmax to select the next token.
  4. Append that token and repeat — \(K\) times total.

This requires \(K\) serial forward passes. Token \(i+1\) cannot be computed until token \(i\) is known, because the model is conditioned on the full prefix.

Memory Bandwidth Bottleneck

Modern LLMs are not compute-bound during inference — they are memory bandwidth-bound. Each forward pass must load the entire model weight matrix from GPU High Bandwidth Memory (HBM) to compute units. For a model with \(N\) parameters in 16-bit precision, each forward pass moves \(2N\) bytes through the memory bus. A single token generation step does \(\mathcal{O}(N)\) multiply-adds but transfers \(\mathcal{O}(N)\) bytes — so the arithmetic units are starved waiting for data.

Consequence: the walltime for one forward pass is roughly constant regardless of batch size (up to a point). Running \(\gamma + 1\) sequences in a batch takes roughly the same time as running 1 — the extra arithmetic is "free" because arithmetic units were idle.

Key Facts — Section 1
  • K tokens requires K serial forward passes. No shortcut without architecture changes.
  • Inference is memory-bandwidth-bound, not compute-bound. Arithmetic units sit idle.
  • Batching \(\gamma+1\) sequences costs roughly the same walltime as batching 1 — this is the foundation of speculative decoding's speedup.

2. Speculative Execution

Speculative execution is a classic CPU optimization: perform a task before you know if it's needed, betting that it will be. The canonical example is branch prediction — the CPU predicts which branch will be taken and begins executing it; if wrong, it discards the work.

The paper generalizes this to the stochastic setting: a task might be needed with some probability \(p\). The core idea applied to LLMs:

  1. Use a cheap draft model \(M_q\) to guess \(\gamma\) likely next tokens.
  2. Run the expensive target model \(M_p\) in parallel on all \(\gamma+1\) possible prefixes.
  3. Accept or reject draft tokens in a way that guarantees the final output distribution equals \(M_p\)'s distribution exactly.

The bet: \(M_q\) is usually right, so \(M_p\)'s single batched pass generates multiple tokens instead of one.

Key Facts — Section 2
  • Speculative execution: do work before you know it's needed, discard if wrong.
  • The draft model guesses; the target model verifies. Output distribution is unchanged.
  • Speedup comes from the target model's batch being nearly "free" compared to running it \(\gamma+1\) times serially.

3. Standardized Sampling

The paper's algorithm only needs to reason about probability distributions \(p(x)\) and \(q(x)\) over the vocabulary. All practical sampling strategies can be cast into this framework by adjusting the logits before softmax:

  • Greedy / argmax: Zero all non-maximum logits, normalize. The resulting distribution is a point mass on the argmax token.
  • Top-k: Zero all but the top-\(k\) logits, normalize.
  • Nucleus (top-p): Zero all tokens outside the smallest set whose cumulative probability exceeds \(p\), normalize.
  • Temperature \(T\): Divide logits by \(T\) before softmax. \(T \to 0\) approaches argmax; \(T \to \infty\) approaches uniform.

After applying any of these transformations, we have a valid probability distribution. The algorithm treats \(p(x)\) and \(q(x)\) as the already-adjusted distributions.

Key Facts — Section 3
  • All sampling strategies reduce to sampling from an adjusted distribution over the vocabulary.
  • \(p(x)\) and \(q(x)\) in the algorithm are post-adjustment distributions, not raw logits.
  • Draft and target must use the same sampling strategy (both use the same temperature, top-k, etc.) for the algorithm to be valid.

4. Algorithm 1 — Line-by-Line

Notation

  • \(M_p\): target model (expensive, the one we want to sample from)
  • \(M_q\): draft model (cheap approximation, same task)
  • \(\gamma\): number of draft tokens to speculate (hyperparameter)
  • \(p_i(x) = p(x \mid \text{prefix} + [x_1, \ldots, x_{i-1}])\): target distribution at position \(i\)
  • \(q_i(x)\): draft distribution at position \(i\)
  • \(\text{norm}(v)\): normalize vector \(v\) to sum to 1
SpeculativeDecodingStep(Mp, Mq, prefix): // Phase 1: Draft model generates γ tokens autoregressively for i = 1 to γ: q_i(x) ← Mq(prefix + [x₁, ..., x_{i-1}]) // draft distribution x_i ~ q_i(x) // sample draft token // Phase 2: Target model evaluates all γ+1 prefixes IN PARALLEL p₁(x), ..., p_{γ+1}(x) ← Mp(prefix), Mp(prefix + [x₁]), Mp(prefix + [x₁, x₂]), ..., Mp(prefix + [x₁, ..., x_γ]) // Phase 3: Rejection loop r₁ ~ U(0,1), ..., r_γ ~ U(0,1) n ← min({ i-1 | r_i > p_i(x_i)/q_i(x_i) } ∪ {γ}) // Phase 4: Adjust distribution (only if a token was rejected) p'(x) ← p_{n+1}(x) if n < γ: p'(x) ← norm(max(0, p_{n+1}(x) - q_{n+1}(x))) // Phase 5: Sample final token and return t ~ p'(x) return prefix + [x₁, ..., x_n, t]

Step-by-Step Explanation

Phase 1 (Draft): The draft model generates \(\gamma\) tokens autoregressively. This is sequential but cheap. Each token \(x_i\) is sampled from \(q_i\), and the draft records the distribution \(q_i(x)\) at each step.

Phase 2 (Parallel target evaluation): The target model is called with a batch of \(\gamma+1\) sequences. The batch is: \[ [\text{prefix}], [\text{prefix}, x_1], [\text{prefix}, x_1, x_2], \ldots, [\text{prefix}, x_1, \ldots, x_\gamma] \] This costs roughly one target forward pass (sequences differ only in length by 1 token, and the memory bandwidth bottleneck means batching is nearly free).

Phase 3 (Rejection): For each draft token \(x_i\), we accept it with probability \(\min(1, p_i(x_i) / q_i(x_i))\). We draw a uniform \(r_i \sim U(0,1)\) and accept iff \(r_i \leq p_i(x_i)/q_i(x_i)\). The variable \(n\) is the index of the first rejection minus one (i.e., \(n\) tokens were accepted). If no rejection, \(n = \gamma\).

Phase 4 (Adjusted distribution): If any token was rejected (\(n < \gamma\)), we sample a correction token from the adjusted distribution \(p'(x) = \text{norm}(\max(0, p_{n+1}(x) - q_{n+1}(x)))\). This compensates for the bias introduced by the draft model having proposed the wrong token. If all tokens were accepted (\(n = \gamma\)), we sample a bonus token from \(p_{n+1}(x)\) — the target distribution after the last accepted draft token.

Why does this produce exactly \(p\)'s distribution? See Section 5. The key insight: the acceptance-rejection scheme ensures each output token follows \(M_p\)'s distribution exactly, regardless of \(M_q\).

Guaranteed minimum: We always produce at least 1 new token per call (the correction or bonus token from \(t \sim p'(x)\)). In the worst case (\(n=0\), first draft token rejected), we get exactly 1 token — same as autoregressive decoding. In the best case (\(n=\gamma\)), we get \(\gamma+1\) tokens.

Key Facts — Section 4
  • Draft phase is sequential and cheap; target phase is one batched call (nearly free).
  • Each step produces 1 to \(\gamma+1\) tokens. Worst case = autoregressive decoding.
  • When a draft token is rejected, all subsequent draft computations for that step are discarded.
  • We always append exactly one token from the target model's distribution — either a correction (if rejected) or a bonus (if all accepted).

5. Speculative Sampling — Full Mathematical Treatment

The Core Claim

We want to sample \(x \sim p(x)\) but only have efficient access to \(q(x)\). Speculative sampling does this:

  1. Sample \(x \sim q(x)\)
  2. Accept \(x\) with probability \(\min\!\left(1, \frac{p(x)}{q(x)}\right)\)
  3. If rejected, sample \(x\) from \(p'(x) = \text{norm}(\max(0, p(x) - q(x)))\)
Claim: The output of this procedure is distributed as \(p(x)\).
▶ Proof (click to expand)

Let \(\alpha = \sum_x \min(p(x), q(x))\) be the probability that a draft sample is accepted. Then:

\[P(\text{accept}) = \sum_x q(x) \cdot \min\!\left(1, \frac{p(x)}{q(x)}\right) = \sum_x \min(q(x), p(x)) = \alpha\]

The output token \(t\) satisfies:

\[P(t = x) = \underbrace{q(x) \cdot \min\!\left(1, \frac{p(x)}{q(x)}\right)}_{\text{draft accepted}} + \underbrace{(1-\alpha) \cdot p'(x)}_{\text{resampled after rejection}}\]

The first term simplifies: \(q(x) \cdot \min(1, p(x)/q(x)) = \min(q(x), p(x))\).

For the second term, \(p'(x) = \frac{\max(0, p(x) - q(x))}{1 - \alpha}\) (normalization constant is \(\sum_x \max(0, p(x)-q(x)) = \sum_x p(x) - \min(p(x), q(x)) = 1 - \alpha\)).

So \((1-\alpha) \cdot p'(x) = \max(0, p(x) - q(x))\).

Therefore: \[P(t = x) = \min(q(x), p(x)) + \max(0, p(x) - q(x))\]

Case 1: \(p(x) \leq q(x)\). Then \(\min = p(x)\), \(\max = 0\). Sum \(= p(x)\). ✓

Case 2: \(p(x) > q(x)\). Then \(\min = q(x)\), \(\max = p(x) - q(x)\). Sum \(= p(x)\). ✓

In both cases \(P(t = x) = p(x)\). \(\blacksquare\)

The DLK Divergence

Definition 3.2 (DLK divergence): \[D_{LK}(p, q) = \sum_x \left|q(x) - M(x)\right| \quad \text{where } M(x) = \frac{p(x) + q(x)}{2}\]
Lemma 3.3: \(D_{LK}(p, q) = 1 - \sum_x \min(p(x), q(x))\)
▶ Proof (click to expand)

\[D_{LK}(p,q) = \sum_x \left|q(x) - \frac{p(x)+q(x)}{2}\right| = \sum_x \frac{|p(x)-q(x)|}{2}\]

\[= \frac{1}{2}\sum_x (p(x) + q(x) - 2\min(p(x),q(x))) = \frac{(1+1) - 2\sum_x\min(p(x),q(x))}{2} = 1 - \sum_x \min(p(x),q(x))\]

\(\blacksquare\)

Corollary 3.4: \(D_{LK}(p,q) \in [0,1]\), symmetric. \(D_{LK}=0 \Leftrightarrow p=q\). \(D_{LK}=1 \Leftrightarrow\) disjoint support.

Theorem 3.5: \(\beta = 1 - D_{LK}(p, q)\)
▶ Proof (click to expand)

\[\beta = \mathbb{E}_{x \sim q}\!\left[\min\!\left(1, \frac{p(x)}{q(x)}\right)\right] = \sum_x q(x) \cdot \min\!\left(1, \frac{p(x)}{q(x)}\right) = \sum_x \min(p(x), q(x))\]

By Lemma 3.3, \(\sum_x \min(p(x), q(x)) = 1 - D_{LK}(p,q)\). \(\blacksquare\)

Corollary 3.6: \(\alpha = \mathbb{E}[\beta] = \mathbb{E}[\min(p, q)]\)

This is the key metric: \(\alpha\) summarizes how well \(M_q\) approximates \(M_p\) across typical prefixes. \(\alpha \to 1\) means the draft is almost always accepted; \(\alpha \to 0\) means speculative decoding degenerates to autoregressive.

Key Facts — Section 5
  • Speculative sampling is exact: output distribution equals \(p(x)\) regardless of \(q(x)\).
  • The adjusted distribution \(p'(x) = \text{norm}(\max(0, p-q))\) compensates for the draft model's bias.
  • \(\alpha = \mathbb{E}[\sum_x \min(p(x), q(x))]\): the expected probability mass where draft and target agree.
  • \(\alpha = 1 - \mathbb{E}[D_{LK}(p,q)]\): higher divergence between models → lower acceptance rate → less speedup.

6. Analysis

Expected Tokens Per Iteration

Let \(\alpha = \mathbb{E}[\beta]\) (mean acceptance rate, assumed i.i.d. per position). In one call to Algorithm 1, the number of accepted draft tokens follows a capped geometric distribution: we accept each token independently with probability \(\alpha\), and stop at the first rejection or at \(\gamma\). Plus one bonus/correction token always.

Let \(N\) = number of accepted draft tokens (before the correction/bonus). Then \(N \in \{0, 1, \ldots, \gamma\}\) with: \[P(N = k) = \begin{cases} \alpha^k (1-\alpha) & k < \gamma \\ \alpha^\gamma & k = \gamma \end{cases}\]

Total tokens per step = \(N + 1\) (draft tokens accepted + 1 correction/bonus). Thus:

Equation 1: \[\mathbb{E}[\text{tokens per step}] = 1 + \mathbb{E}[N] = \frac{1 - \alpha^{\gamma+1}}{1 - \alpha}\]
▶ Derivation (click to expand)

\[\mathbb{E}[N] = \sum_{k=0}^{\gamma-1} k \cdot \alpha^k(1-\alpha) + \gamma \cdot \alpha^\gamma\]

Let \(S = \sum_{k=0}^\gamma k \cdot \alpha^k (1-\alpha)\) (geometric series derivative):

\[\sum_{k=0}^{\gamma} \alpha^k = \frac{1 - \alpha^{\gamma+1}}{1-\alpha}\]

\[\mathbb{E}[N+1] = \mathbb{E}[N] + 1 = \sum_{k=0}^{\gamma}(k+1) P(N=k) = \sum_{j=1}^{\gamma+1} j \cdot P(N=j-1)\]

Equivalently: \(\mathbb{E}[N+1] = \sum_{k=0}^\gamma P(N \geq k) = \sum_{k=0}^\gamma \alpha^k = \frac{1-\alpha^{\gamma+1}}{1-\alpha}\). \(\blacksquare\)

Walltime Improvement

Definition 3.7 (Cost coefficient): \(c = \frac{\text{walltime of one } M_q \text{ run}}{\text{walltime of one } M_p \text{ run}}\)

In practice, \(M_q\) is orders of magnitude smaller than \(M_p\), so \(c \ll 1\) (often \(c < 0.05\)).

Theorem 3.8 (Walltime speedup): \[\text{speedup} = \frac{1 - \alpha^{\gamma+1}}{(1-\alpha)(\gamma c + 1)}\]
▶ Proof (click to expand)

Let \(T\) = walltime of one \(M_p\) forward pass. Each call to Algorithm 1 costs:

  • \(\gamma c T\): running \(M_q\) for \(\gamma\) steps
  • \(T\): one batched \(M_p\) call (the \(\gamma+1\) parallel evaluations cost ~1 pass due to batching)

Total cost per step: \((\gamma c + 1)T\). Expected tokens per step: \(\frac{1-\alpha^{\gamma+1}}{1-\alpha}\).

Cost per token with Algorithm 1: \(\frac{(\gamma c + 1)(1-\alpha)}{1-\alpha^{\gamma+1}} T\).

Cost per token with autoregressive: \(T\). Speedup = ratio. \(\blacksquare\)

Corollary 3.9: If \(\alpha > c\), there exists \(\gamma > 0\) giving speedup. Minimum speedup (at \(\gamma=1\)) is \(\frac{1+\alpha}{1+c}\).

Arithmetic Operations

Theorem 3.11: The expected factor increase in arithmetic operations is: \[\frac{(1-\alpha)(\gamma\hat{c} + \gamma + 1)}{1 - \alpha^{\gamma+1}}\] where \(\hat{c}\) = ratio of FLOPs per token for \(M_q\) vs \(M_p\).

Note: walltime speedup and arithmetic savings are different! Speculative decoding can increase total FLOPs slightly (rejected drafts waste compute) while still improving walltime (because of the memory bandwidth bottleneck).

Optimal γ

Theorem 3.8 gives speedup as a function of \(\gamma\). Maximizing over \(\gamma\) (treating \(\gamma\) as continuous, differentiating, setting to zero) gives an implicit equation. In practice, the optimal \(\gamma\) is found empirically by sweeping values of \(\gamma\) and measuring walltime.

Worked example: \(\alpha = 0.75\), \(c = 0.03\).

  • At \(\gamma=4\): speedup \(= \frac{1-0.75^5}{(1-0.75)(4 \times 0.03 + 1)} = \frac{1-0.237}{0.25 \times 1.12} \approx \frac{0.763}{0.280} \approx 2.73\times\)
  • At \(\gamma=7\): speedup \(= \frac{1-0.75^8}{(0.25)(7 \times 0.03 + 1)} \approx \frac{0.900}{0.303} \approx 2.97\times\)
  • At \(\gamma=12\): speedup \(\approx \frac{0.968}{0.340} \approx 2.85\times\) — diminishing returns, overhead grows
Key Facts — Section 6
  • \(\mathbb{E}[\text{tokens/step}] = (1-\alpha^{\gamma+1})/(1-\alpha)\). Derivation: sum of probabilities of \(N \geq k\).
  • Walltime speedup = (expected tokens per step) / (normalized step cost). This factors as Theorem 3.8.
  • Condition for speedup: \(\alpha > c\) (draft quality must exceed its relative cost).
  • Speculative decoding may use more total FLOPs but still be faster (memory bandwidth is the bottleneck, not FLOPs).
  • Optimal \(\gamma\) balances increased expected tokens vs. increased draft overhead.

Key observations:

  • Summarization has higher \(\alpha\) than translation because there is less creative freedom — the draft and target tend to agree more.
  • The speedup is not as high as \(\mathbb{E}[\text{tokens/step}]\) suggests because of infrastructure overhead (communication, batching, attention mask handling).
  • The method requires no retraining or architecture changes to either model.
  • In memory-bandwidth-constrained settings (single accelerator), speedups are higher. In compute-constrained settings (large batch inference), speedups are lower.
Key Facts — Section 7
  • Real-world α ≈ 0.7–0.8 for tasks where draft and target share training distribution.
  • Observed speedups: 2–3x on T5-XXL, consistent with Theorem 3.8 predictions.
  • Higher α on tasks with less creative freedom (summarization > translation > open-ended generation).
